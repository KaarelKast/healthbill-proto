# Phase 1: Clarify Requirements

Use `AskUserQuestion` to ask 3–5 focused questions before writing any code. Cover:

- The exact expected behavior and any UI/UX details
- Edge cases and error states to handle
- Constraints (performance, accessibility, specific patterns to follow or avoid)
- Integration points: which existing components, hooks, or API methods are involved
- Definition of done: how will we know the implementation is correct?

**Do not mark this phase complete and do not move to Phase 2 until the user has answered all questions.**
